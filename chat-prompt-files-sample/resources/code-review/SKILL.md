# Code Review Skill

This skill provides capabilities for reviewing code and providing constructive feedback.

## Capabilities

- Review code for best practices
- Identify potential bugs and issues
- Suggest improvements for readability
- Check for security vulnerabilities
- Evaluate test coverage

## Guidelines

When performing code reviews:

1. Be constructive and specific in feedback
2. Explain the reasoning behind suggestions
3. Prioritize issues by severity
4. Acknowledge good patterns when found
5. Consider the context and constraints

## Review Checklist

- [ ] Code follows project conventions
- [ ] Error handling is appropriate
- [ ] Edge cases are considered
- [ ] Code is readable and maintainable
- [ ] Tests cover critical paths
